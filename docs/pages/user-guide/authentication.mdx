# Authentication

## Generate Your API Key

You have two options to generate your API key:

### Option A: Using the CLI

AiMo Network uses cryptographically signed API keys linked to your Solana wallet:

```bash
# Generate an API key valid for 90 days
aimo keygen --tag dev --valid-for 90 --scopes completion_model

# This will output an API key like:
# aimo-sk-dev-[base58-encoded-key]
```

**API Key Parameters:**

- `--tag`: Environment tag (use "dev" for development, "prod" for production)
- `--valid-for`: Validity period in days (default: 90 days)
- `--scopes`: Permissions for the key (currently supports "completion_model")
- `--usage-limit`: Optional usage limit (0 = unlimited)

### Option B: Using the Dashboard

Alternatively, you can generate API keys and manage your account through our web dashboard.

## API Key Management

### Security Best Practices

1. **Keep Keys Secure**: Store API keys securely, never expose them in client-side code
2. **Use Environment Variables**: Store keys in environment variables or secure configuration
3. **Rotate Keys Regularly**: Generate new keys before expiration
4. **Monitor Usage**: Track your API key usage and costs

### Key Revocation

If your API key is compromised, you can revoke it:

```bash
# Sign a revocation message with your Solana wallet
solana-keygen verify [signature] [message] --keypair ~/.config/solana/id.json

# Submit revocation through the API (contact node operator for details)
```

### Key Expiration

API keys have expiration dates. Before your key expires:

1. Generate a new key with `aimo keygen`
2. Update your applications with the new key
3. The old key will stop working after expiration
