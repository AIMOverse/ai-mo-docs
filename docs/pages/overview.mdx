# Quick Start

Get up and running with AiMo Network in just a few minutes.

## What You'll Learn

This quick start guide will help you:
- Set up your Solana wallet
- Generate your API key using the CLI
- Make your first inference request
- Understand the decentralized payment system

## Prerequisites

- Basic understanding of AI/ML models and APIs
- Command line environment
- Internet connection
- Solana CLI tools (for wallet and payment management)

## Step 1: Set Up Your Solana Wallet

AiMo Network uses Solana for decentralized payments:

1. **Install Solana CLI**: Follow [Solana's installation guide](https://docs.solana.com/cli/install-solana-cli-tools)
2. **Generate a wallet**: 
   ```bash
   solana-keygen new
   ```
3. **Fund your wallet**: You'll need Solana tokens for the escrow account

## Step 2: Install AiMo CLI

Install the AiMo CLI tool for key generation and account management:

```bash
cargo install aimo-cli
```

## Step 3: Generate Your API Key

Generate a cryptographically signed API key linked to your Solana wallet:

```bash
# Generate an API key valid for 90 days
aimo keygen --tag dev --valid-for 90 --scopes completion_model
```

Your API key will look like this:
```
aimo-sk-dev-[base58-encoded-key]
```

## Step 4: Set Up Your Escrow Account\

For paid models, you need to deposit USDC into your escrow account:

1. **Go to the dashboard**: Visit [https://aimo.network/demo/dashboard](https://aimo.network/demo/dashboard)
2. **Top up your account**: Add USDC to your escrow balance using the dashboard interface
3. **Automatic payments**: The system will handle payments automatically based on usage

:::note
The escrow system locks estimated costs before requests and settles actual usage after completion.
:::

## Step 5: Make Your First Request

### Using cURL

```bash
curl -X POST "https://devnet.aimo.network/api/v1/chat/completions" \
  -H "Authorization: Bearer aimo-sk-dev-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "8W7X1tGnWh9CXwnPD7wgke31Gdcqmex4LapJvQ2afBUq:deepseek-chat-v3",
    "messages": [
      {"role": "user", "content": "Hello! Can you help me understand AiMo Network?"}
    ],
    "stream": false,
    "max_tokens": 50
  }'
```

### Using Python

```python
import requests
import json

url = "https://devnet.aimo.network/api/v1/chat/completions"
headers = {
    "Authorization": "Bearer aimo-sk-dev-[your-api-key]",
    "Content-Type": "application/json"
}

data = {
    "model": "8W7X1tGnWh9CXwnPD7wgke31Gdcqmex4LapJvQ2afBUq:deepseek-chat-v3",
    "messages": [
        {"role": "user", "content": "Hello! Can you help me understand AiMo Network?"}
    ],
    "stream": False,
    "max_tokens": 50
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
```

## Step 6: Finding Model Providers

To use AiMo Network, you need to find available models and their provider addresses:

1. **Visit the discover page**: Go to [https://aimo.network/demo/discover](https://aimo.network/demo/discover) to browse available models
2. **Copy the model slug**: Find the model you want to use and copy its complete slug in the format `address:model_name`
3. **Use in your requests**: Paste the copied slug directly into your API requests

Example model slugs you might find:
- `8W7X1tGnWh9CXwnPD7wgke31Gdcqmex4LapJvQ2afBUq:deepseek-chat-v3`

```python
response = requests.post("https://devnet.aimo.network/api/v1/chat/completions",
```

## Understanding Costs and Payments

AiMo Network uses a decentralized payment system with automatic escrow:

### How Payments Work

1. **Pre-Authorization**: Before each request, estimated cost is locked in your escrow
2. **Actual Settlement**: After completion, you're charged only for actual token usage
3. **Automatic Recovery**: Failed requests automatically recover locked tokens

### Response Format

Each request returns usage information for billing:

```json
{
  "choices": [...],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 75,
    "total_tokens": 100
  }
}
```

### Payment Calculation

- **Token Usage**: Based on input tokens (prompt) and output tokens (completion)
- **Provider Pricing**: Each provider sets their own pricing per token
- **Blockchain Settlement**: Payments are settled automatically on Solana

## Error Handling

Always implement proper error handling for common issues:

```python
import requests

try:
    response = requests.post("https://devnet.aimo.network/api/v1/chat/completions",
        headers={"Authorization": "Bearer aimo-sk-dev-[your-key]", "Content-Type": "application/json"},
        json={
            "model": "provider_pubkey:gpt-4",
            "messages": [{"role": "user", "content": "Hello"}]
        }
    )
    
    if response.status_code == 200:
        data = response.json()
        print(data['choices'][0]['message']['content'])
    elif response.status_code == 401:
        print("Error: Invalid or expired API key")
    elif response.status_code == 402:
        print("Error: Insufficient balance in escrow account")
    elif response.status_code == 404:
        print("Error: Provider or model not found")
    else:
        print(f"Error {response.status_code}: {response.text}")
        
except Exception as e:
    print(f"Network error: {e}")
```

## Best Practices

1. **Store API keys securely** - Use environment variables, never expose in code
2. **Handle errors gracefully** - Implement retry logic with exponential backoff
3. **Monitor escrow balance** - Keep sufficient tokens for your usage patterns
4. **Use correct model format** - Always use `provider_pubkey:model_name` format
5. **Implement streaming** - Use streaming for better user experience with longer responses

## Troubleshooting

### Common Issues

**Authentication Error (401)**
- Verify your API key format: `aimo-sk-dev-[base58-encoded-key]`
- Check that your API key hasn't expired
- Ensure it's included in the Authorization header

**Insufficient Balance Error (402)**
- Your escrow account has insufficient tokens
- Top up your escrow account with more tokens
- Contact your node operator for payment token details

**Provider/Model Not Found (404)**
- Verify the provider's Solana public key is correct
- Ensure you're using the format: `provider_pubkey:model_name`
- Check with your node operator for available providers

**Failed Dependency (424)**
- Your escrow account isn't set up properly
- Set up your escrow account with the required payment tokens
